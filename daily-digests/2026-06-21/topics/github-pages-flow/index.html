GitHub Pages 发布流程

今天把每日复盘从本地页面变成正式发布页，关键不在“写一个 HTML”，而在把发布链路变成可重复、可验证、可停止的流程。

触发用户明确要求每日复盘发布到个人 GitHub Pages，并要求首页 Recent Posts 可见。
边界发布仓库必须先检查干净；如果有无关脏改动，停止并汇报。
产物总账页、主题页、专题页，以及 assets/posts.js 文章索引。
确认仓库目标仓库是 Think2twice.github.io，当前分支是 master，远端是 GitHub Pages 仓库。
检查状态运行 git status --short --branch。本次发布仓库干净，可以继续。
写入页面每日总账在 daily-digests/2026-06-21/，主题/专题在同日子目录下。
更新索引在 assets/posts.js 新增正式复盘文章，让首页和归档能发现。
本地验证检查 HTML 存在、JS 语法可解析、敏感信息扫描无明显命中、Git 只暂存本次文件。
提交推送提交到 origin master 后，再打开线上 URL 做访问验证。

规则候选：每日复盘发布以后不应该只检查“页面文件存在”，还要检查 posts.js、首页可见性、归档可见性、敏感信息、线上 URL。
