Razer Mac 快捷键映射

补充页侧重“Windows 板载宏”和“Mac 端输入翻译”的关系。

问题抽象

Windows 里能给鼠标写入 `Ctrl+C / Ctrl+V`，但 Mac 应用需要 `Command+C / Command+V`。如果 Mac 版厂商软件不支持当前设备，就把鼠标当作会发出输入事件的外设，在 Mac 端做翻译。

1. 保留板载宏
Windows 继续写 `Ctrl+C / Ctrl+V`。
2. 查设备事件
用 EventViewer 看 Mac 收到键盘事件、鼠标按钮还是水平滚动。
3. 按设备映射
优先系统修饰键；不够再用 Karabiner。
4. 应用验证
在真实编辑器里确认复制粘贴，不影响主键盘。

结论

这类问题的关键不是让 Windows 认识 Command，而是把“鼠标发出的 Ctrl”与“Mac 应用需要的 Command”拆成两层处理。
