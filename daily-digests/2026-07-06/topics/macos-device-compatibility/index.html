Mac 外设兼容与按键翻译

从“雷云不识别鼠标”抽象出一套可复用排查法：系统识别、厂商支持、输入事件、按设备映射。

当日流水来源

  • macOS 已识别 Razer Basilisk V3，说明线材、USB 口和基础 HID 能力大概率正常。
  • 普通 Basilisk V3 不在当时 Mac 版配置软件支持列表里，所以高级配置不能用软件接管。
  • Windows 里已保存的 `Ctrl+C` / `Ctrl+V` 宏，可以在 Mac 侧按设备翻译成 `Command+C` / `Command+V`。
1. 系统识别
先查 macOS 设备树或系统信息，确认 Product Name、Vendor ID、Product ID。
2. 厂商支持
再查厂商当前支持列表，区分普通型号、Pro、35K 等细分版本。
3. 事件翻译
最后用系统修饰键、Karabiner、BetterTouchTool 或 SteerMouse 按设备映射。

高价值结论

不要把“配置软件不显示”直接等同于“设备坏了”。设备可能已经作为普通 HID 正常工作，只是厂商软件没有支持这个具体型号。跨系统快捷键也不一定要在 Windows 软件里解决，可以保留板载 `Ctrl` 宏，在 Mac 端把这只设备发出的 Control 解释成 Command。

下次验证

用事件查看工具拨一下滚轮左/右，确认 Mac 收到的是 `ctrl+c`、`ctrl+v`，还是鼠标按钮或水平滚动。只有确认事件类型后，再选择系统修饰键映射、Karabiner 或鼠标专用工具。
