408 同步与发布门禁

这组线程说明：408 学习站已经不是单纯“把页面发出去”。发布链路必须同时照顾四科内容同步、题库资源、做题记录、构建入口、部署连接和线上验收。

今日证据

  • 凌晨修复：四科做题记录同步白名单补齐，账号菜单和个人页统计同步更新，测试与部署通过。
  • 晚间 worker：多次在构建阶段遇到资源或入口瞬时缺失，失败后文件实际存在。
  • 另一个 worker：构建和链接检查通过，但部署阶段远程连接关闭，不能把旧线上可访问当作本轮成功。

稳定门禁

触发
任何正式站发布、四科同步或题库修复。
动作
冲突扫描、JSON 解析、sync、check:links、deploy:phase4 串行执行。
验证
部署成功后抽样章节页、题库资源、同步脚本和做题记录。

后续规则候选

晚间 worker 需要运行锁或 single-flight。构建前应确认生成目录稳定，避免多个 worker 同时改写 public 或 dist。做题记录同步应与页面发布同级验收。
