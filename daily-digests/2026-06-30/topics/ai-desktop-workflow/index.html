AI 云桌面与 WorkBuddy 使用边界

把“远程 Windows 云桌面、WorkBuddy、OpenClaw、Codex”拆成四个角色，避免把后台控制台误当成助手本体。

返回总账

触发

Fay 在云桌面控制台里看到桌面池、云桌面、WorkBuddy 相关入口，不确定自己应该在哪里操作。

结论

云桌面后台负责购买、开机、重启、连接；真正使用 WorkBuddy 要先进入远程 Windows 桌面。WorkBuddy 更偏电脑操作和办公自动化，Codex 更偏本机项目、代码、文档和命令执行。

动作路径

先看云桌面管理里的实例状态，等可用后下载客户端，登录并连接远程 Windows，再在远程桌面里打开 WorkBuddy / OpenClaw。不要在桌面池管理里反复创建新资源。

可复用提醒

看到“云桌面控制台”时，先分三层：资源后台、远程电脑、电脑里的 Agent。后台不是 Agent 本体。

公开版本已脱敏：不写真实服务器资产、账号、平台凭据、私密聊天原文和不必要本机路径。
