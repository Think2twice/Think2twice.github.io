408 发布门禁

晚间 worker 的边界很窄：白天内容已经写进源文件，夜间只同步、构建、检查和发布。今天的结果仍是“发布未完成”。

可确认状态

  • 题库 JSON 解析与冲突标记扫描有通过记录。
  • npm run sync 有一次生成目录清理瞬时失败，重试后成功同步 4 科 26 章。
  • npm run check:links 有成功记录，也有并行线程里的瞬时构建入口失败记录。
  • npm run deploy:phase4 进入发布链路后仍被本地 dist 瞬时遍历或 SSH/rsync 阶段阻断。

不能算成功的原因

正式站旧版本可访问、首页返回 200、旧 Last-Modified 存在，都不能证明本轮新内容已经发布。只有 deploy 完成并读回代表页面，才算通过。

下一步

  1. 给晚间发布加 single-flight，避免多个 worker 同时写同一份 dist。
  2. 把 dist 瞬时 ENOENT/ENOTEMPTY 做成固定排障分支。
  3. 若 SSH/rsync 再失败，先检查本机网络和 Clash Verge 的 TUN/虚拟网卡模式。
  4. 最后只按正式入口和代表页读回验收。
