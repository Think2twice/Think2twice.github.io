失败分型

  • 瞬时 `check:links` 或 dist 路径失败：更像并发/生成目录状态问题，不能直接部署。
  • SSH/rsync 失败：发布通道问题，优先查本机 TUN/虚拟网卡。
  • 旧站 200：旧版本存活，不是本轮成功。

后续动作

先建立 single-flight 锁，再由一个 worker 重跑 `sync -> check:links -> deploy:phase4 -> 线上代表页验收`。如果 SSH 再失败，先处理网络通道，不改教材内容。
