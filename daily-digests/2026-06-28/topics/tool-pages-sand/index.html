工具与 Pages 散沙：试跑、发布、验收

今天的小工具和 Pages 工作都不算大项目，但它们有共同点：先定义目标场景，再跑真实输出，再用结果判断工具是否值得继续投入。

Aliens_eye 试跑结论

它是什么

Python 命令行用户名扫描工具，可以本地安装或 Docker 运行，输出 JSON、CSV、HTML、Markdown 报告。它不是需要部署的网页服务。

它不是什么

它不是“找一个普通人真实社交账号”的可靠系统。中文平台、封闭社交、反爬和同名误判会让结果噪声很大。

适合场景

开发者 handle、开源社区用户名、品牌名占用、粗略 OSINT 候选列表。

不适合场景

找国内真实社交轨迹、证明某账号属于某个人、直接把 Found 当证据。

Pages 课程返工规则

  1. 教程页不能只有目录卡片，正文要能解释“为什么这么做”。
  2. 每节至少包含工程背景、关键文件、操作流程、验证门禁、常见坑和练习。
  3. 长课程必须有上一节、目录、下一节，不让读者每次回目录重进。
  4. 发布后不只看 Git 提交，还要在线验证新版标志、页面高度、导航和 200 状态。

今天可以沉淀的小规则

判断工具是否“垃圾”之前，先把场景说清楚：如果场景是中文真实社交账号追踪，Aliens_eye 很弱；如果场景是英文开发者 handle 粗筛，它还有用。工具评估不看功能列表长度，先看目标人群和数据源是否匹配。
