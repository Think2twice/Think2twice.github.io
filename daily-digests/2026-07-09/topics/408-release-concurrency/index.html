408 发布并发门禁

同一晚 408 发布 worker 出现了三类结果：构建阶段失败、rsync 阶段失败、最终成功发布。日报不能只写其中一条，要把最终可用和系统风险同时入账。

可确认成功

  • 有 worker 完成同步、链接检查、部署。
  • 正式入口首页与代表页面返回 200。
  • 本轮内容包含四科同步产物、题库和若干章节资源。

仍有风险

  • 其他 worker 停在 VitePress 临时模块或 rsync exit 23。
  • rsync 失败后样例文件本地又存在，说明更像构建目录或并发状态扰动。
  • 旧首页 200 不能当成任意 worker 成功的证据。

规则候选

下轮发布前先做 single-flight：排除自身匹配，确认没有活跃 build/rsync/SSH 发布进程，再进入 sync、check、deploy。收尾只写一个最终判定，并保留失败 worker 作为风险材料。
