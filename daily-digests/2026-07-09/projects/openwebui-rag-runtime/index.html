OpenWebUI RAG 运行态

今天 OpenWebUI 的关键不是新增 UI，而是把文件读取链路从“provider 失败就整轮失败”改成“正文先可用，索引后台补偿”。

问题抽象

长 PDF 已经抽取出正文，但 embedding/provider 网络失败会拖死整个文件处理。用户实际需要的是先能围绕文件正文提问，而不是被向量索引失败挡住。

处理方式

  • 普通聊天上传文件：正文抽取成功后，即使向量保存失败，也标记为当前对话可用。
  • 知识库 collection：仍保持严格，不把失败伪装成检索成功。
  • 后台补索引：按退避节奏重试，成功后写回索引状态。
  • 配置边界：Sub2API 管理台账号测试和 OpenWebUI RAG 直连是两套配置，provider base URL 变化时两边都要核对。

验证口径

服务器健康、函数级 embedding smoke 和配置 grep 只能说明运行态修复已上线；最终完成仍要 Fay 在真实 UI 上传大 PDF 并直接提问确认。
