OpenWebUI 源码迁移与私有产品化

当天最大的工程主线：从商业级开源页面选型，走向本地源码、服务器回填、Skill 和未来部署替换。

阅读方式

这是从每日流水账抽出的脱敏方法页，只保留结构、触发条件、动作和验证。

项目主线

根目标

把 OpenWebUI 从“找一个像 ChatGPT 的商业级开源页面”推进成 Fay 自己可控、可部署、可扩展的私有 AI 工作台。

工程层

当前工程链路是：开源成品观摩 -> 服务器 Docker 热补丁 -> 本地源码 fork -> 进度表驱动 -> 图片/文件/RAG/网关/模型体验 -> 拆提交 -> 源码部署替换旧服务。

认知层

“像 ChatGPT”不是一层 UI，而是模型路由、文件系统、RAG、图片、语音、权限、缓存、附件、状态卡和部署数据卷共同拼出来的产品体验。

今天的关键决策

文件生成

不把 /mnt 或 Markdown 假链接当完成，必须落到真实可下载附件。

RAG

默认质量优先；维度验收必须匹配 large，不用省钱版本冒充默认。

Skill

OpenWebUI 项目需要单一 Skill 和 references，帮助新线程接手，不再反复口述。

下一步

  1. 清理工作区并按功能域拆提交。
  2. 选一个服务器热补丁完整迁回本地源码。
  3. 跑普通用户视角端到端验收。
  4. 把验收证据写回 PROJECT_PROGRESS 和 HANDOFF。
