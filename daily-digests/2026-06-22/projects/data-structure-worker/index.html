大石头专题 / 数据结构自动化

从“写 README”升级到有质量门槛的 worker

6 月 21 日的数据结构主线不是某个排序知识点，而是一套 guard-first、单任务、动画和专题硬门槛的自动化工作流。

核心判断

当前已经进入“自动化生产 + 质量门槛”阶段，下一步要治理触发频率、锁冲突和站点接入。

7.x查找章节补救与图谱
8.1-8.7排序章节连续推进
3.x/4.x夜间接力扩展
guard锁占用硬停止

工作流骨架

1. 门卫
第一条项目命令只运行 `automation_guard.py start`；`stop` 立即退出，`run` 才读控制文件。
2. 唯一目标
由 `rebuild-progress.json` 与队列共同确认一个 content 小节或图谱任务；完成后停止。
3. 讲解链
README 承载主讲义，必要时补专题；动态过程要 SVG + 可控交互动画。
4. 题库回流
题目写 section、sourceSlice、reviewLinks；不确定和污染片段进 diagnostics。
5. 验证
JSON、SVG XML、JS 控件、Markdown 链接、残片扫描、heartbeat/release 都要确认。

阶段判断

这套 worker 已经不只是“自动生成教材”，而是在训练一个闭环：资料切片、讲义、专题、动画、题库、图谱、控制文件、停止条件。风险也更清楚：触发频率高时，stale lock 和空线程会污染流水账；下一步应把锁健康度作为调度指标。
