浏览器用户脚本

LinkSwift 被确认是浏览器用户脚本，不是服务器部署项目；重点是脚本管理器和 Chrome 用户脚本权限。

部署散沙

服务器应用部署可复盘成“配置注入 -> 服务启动 -> 反代 -> HTTPS -> 登录/注册策略验证”，公开页面不保留真实入口和账号。

GitHub Pages

本轮确认发布仓库为 `Think2twice.github.io`，master 根目录发布，`assets/posts.js` 是首页/归档入口。

散沙

HTML 设计规则

每日页继续沿用浅色纸面、克制网格、短卡片和清晰表格，不使用炫技型大 hero。

可复用检查单

浏览器脚本
先判断是不是用户脚本；再确认脚本管理器、Chrome 用户脚本权限、目标网页是否注入按钮。
部署记录
公开页只写部署链路：配置注入、服务状态、反代、HTTPS、登录/注册验证；真实入口留在私有工作记录。
GitHub Pages
发布前先看 `git status --short`，只暂存本日目录和 `assets/posts.js`，发布失败不改发其他站。
HTML 风格
教程库继续使用浅色纸面风格，第一屏直达内容，不做营销 hero。
