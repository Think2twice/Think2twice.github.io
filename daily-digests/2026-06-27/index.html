今日高价值流程

GitOps-lite 接手流程

触发：OpenWebUI 要改功能、部署或资源包。

动作：确认权威源码、构建产物、服务器部署面和数据卷，再改。

验证：本地构建、服务器运行、配置/DB 状态和文档记录同时对齐。

服务器清理流程

触发：新服务已上线，旧资产占空间。

动作：先确认当前运行服务和数据卷，再删旧容器、旧镜像、旧缓存。

验证：正式入口仍可访问，关键数据卷保留，磁盘空间下降。

工具扩充门禁

触发：用户要求多接 Skill、工具、函数或 MCP。

动作：先分类：Prompt、Skill、Tool、Action、Filter、Pipe、外部 MCP。

验证：高风险项默认禁用，普通用户可见权限正确，不重复创建。

学习站首页修复

触发：总站入口偏科、暗色模式可读性差。

动作：四科入口并列，颜色区分，暗色模式单独检查。

验证：构建、链接检查、线上首页和 CSS 均确认。

Git / GitHub / GitHub Pages

本轮继续按每日复盘发布例外执行：生成 Pages 目录和本地留档，更新 `assets/posts.js`，每日条目保持 `featured:false`、`homeVisible:false`。Archive 读取完整文章列表，首页只读过滤后的公开教程。

Skill / 自动化

OpenWebUI 项目 Skill 被进一步写入 Source of Truth 和工具默认状态规则。每日自动化继续保持“先流水、再聚合、再全景、再认知替换”的顺序。
