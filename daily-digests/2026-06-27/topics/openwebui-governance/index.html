OpenWebUI 治理散沙合集

这组不是单一功能，而是同一天围绕源码真相、资源包、移动端、文件桥接和服务器清理反复暴露出的系统治理问题。

今天沉淀出的规则

四层先对齐

任何改动先确认：权威源码、构建产物、服务器部署面、运行数据卷。否则容易把临时容器、旧部署和真实服务混在一起。

资源默认不污染

Prompts、Skills、Tools 可以丰富，但全局 Action、Filter、Pipe、外部 MCP 都要有默认状态和回滚策略。

复用流程

移动端问题先做设备矩阵

同一问题在 iPhone Safari 复现、三星浏览器正常时，应按 WebKit 事件链处理，而不是继续泛化 CSS 补丁。

清理服务器先保当前服务

先确认当前运行服务、镜像和数据卷，再处理旧容器、旧目录、旧缓存和旧镜像。

验证方式

文件桥接看三层证据

代码路径是否绕过本地解析、payload 是否仍带原文件、服务端日志是否看到附件。

隐藏任务看运行态

标题生成、标签、追问、搜索 query、自动补全、图片 prompt、automation 要逐项确认，而不是笼统说后台任务开关。
