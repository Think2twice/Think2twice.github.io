工具系统边界散沙合集

Agent 邮箱、AList 缓存、Action Functions、MCP 候选配置都说明：工具系统不是越多越好，而是默认边界要清楚。

工具分类

Agent 邮箱

它是收验证码、验证链接和通知的收件箱，不是自动注册任何网站的通行证。

Action Functions

适合做“点击触发”的辅助动作；如果变成全局自动介入，就会污染每条回答。

默认状态

外部 MCP 先候选禁用

没有审过来源、权限和回滚前，不应直接给用户启用未知公网 MCP。

AList 缓存先备份再调

目录缓存变长能减少反复转圈，但会带来新增/删除文件延迟可见的代价。

验证方式

权限可见性

普通用户能看到该看的资源，不能看到危险入口。

速度与新鲜度

缓存策略同时记录命中速度和文件更新延迟，避免只追求快。
