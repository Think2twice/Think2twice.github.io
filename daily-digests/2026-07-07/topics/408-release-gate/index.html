408 发布门禁

这个主题记录同一晚多个发布 worker 暴露的边界：本地同步和构建能过，不代表正式站已经发布；旧版本可访问也不代表本轮成功。

sync
四科内容同步进入统一站点。
check
构建、链接和 legacy 资源检查最终可过。
deploy
SSH/TUN 网络层超时，rsync 未完成。
accept
未完成本轮线上验收，旧站 200 不能算成功。

保留下来的规则

  • 发布失败要写清停止阶段。
  • 本地门禁通过后仍要单独看部署通道。
  • 线上验收必须证明新内容更新，不只看入口存活。
  • 同一发布目标应加 single-flight 锁，避免多个 worker 同时写输出目录。
