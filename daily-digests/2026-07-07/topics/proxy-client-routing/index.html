国内 App 异常怎么判断

如果 Google/YouTube 能走代理，但抖音、网易云、国内服务异常，优先怀疑客户端分流：直接导入“节点列表”不等于导入“完整规则配置”。

  • Shadowrocket：全局路由应使用配置/规则模式，避免国内流量全进代理。
  • Clash/Mihomo：用 Rule 模式，并确认国内媒体、全球直连等策略组指向 DIRECT。
  • x-ui：服务端 JSON 订阅可补直连规则，但普通节点订阅仍需要客户端规则。

协议定位

  • VLESS REALITY Vision：适合做直连长期基线，但不能套普通 CDN。
  • Hysteria2/TUIC：适合 UDP/QUIC 速度补充，客户端支持与网络环境要实测。
  • VLESS/WS + CDN：适合隐藏源站或兼容旧客户端，不是速度优先解法。

验收标准

一个节点要同时通过：Google 204、YouTube Stats、国内 App 直连表现、订阅字段稳定、客户端模式正确。只看绿色测试不足以收口。
