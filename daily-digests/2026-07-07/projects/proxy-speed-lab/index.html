Proxy Speed Lab 路线切换

这个专题记录 2026-07-07 的大石头：目标不是找一个“能打开”的节点，而是把用户端到节点入口这段链路测准、分型、收束。

已经排除

  • HY2 测试：服务端可通，但 Shadowrocket 和本机客户端超时。
  • Reality 直连：公网直连多个端口超时，未进入协议参数阶段。
  • 免费 Cloudflare/ProxyIP 大池：真实吞吐远低于目标。

保留判断

  • 旧服务器出口很快，源站不是主要瓶颈。
  • 真正要优化的是用户网络到入口。
  • “绿色测试”不等于 YouTube 高吞吐。

本地资产

  • 候选 IP 扫描脚本：只临时生成测试配置，不直接写入正式订阅。
  • 测速结果文档：记录协议、候选池和吞吐结论。
  • 新 VPS 验收脚本：到手后先跑本地连通、远端出口、下载吞吐和内核基线。
  • 购买档案：作为备选，不在“一分钱不花”的当前链路里继续推进。

当前零成本主线

  1. 保留现有可用 WS/CDN 节点。
  2. 修客户端分流和订阅字段。
  3. 服务端只做可回滚的 Nginx WS 长连接参数与 TCP 缓冲优化。
  4. 最后用 Fay 手机/电脑真实链路验收。
