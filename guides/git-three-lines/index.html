本地 Git / Git Worktree / GitHub 复读手册

把 Git 看成三条轨道，就不容易忘。

本地 Git 管你电脑上的版本，Git worktree 管并行任务文件夹，GitHub 管云端分支和 PR。commit、stash、merge、push 都只是这三条轨道上的动作。

先看三条线 打开操作沙盘

第一件事：分清三条线

你看到的“分支”可能在本地，可能在 GitHub，也可能只是某个 Git worktree 当前 checkout 的分支。

Line 1

本地 Git

你电脑上的项目文件夹和隐藏的 `.git/` 历史。`commit` 只是把改动保存到本地历史，还没有上传。

Line 2

Git worktree

同一个仓库历史开出多个本地工作区。一个任务一个文件夹，常见分支名像 `feature/phase4-email-autosync`。

Line 3

GitHub remote

云端仓库、远程分支、Pull Request。只有 `push` 后，GitHub 才能看到你的本地 commit。

基础流程：从改文件到 GitHub

把 Git 想成四个空间：工作区、暂存区、本地仓库、远程仓库。

01 edit

改文件

你正在编辑的是 working tree。这里的内容还没有进入 Git 历史。

02 add

暂存

`git add` 表示“下一次 commit 要包含这些改动”。

03 commit

本地存档

`git commit` 在本地创建一个版本点。它不等于上传。

04 push

上传远端

`git push` 才把本地 commit 发到 GitHub 的远程分支。

05 PR

请求合并

PR 是 GitHub 页面，用来请求把 head branch 合进 base branch。

你最该记住的一句

commit 是本地保存，push 才到 GitHub。

所以截图里显示某个 `feature/...` 分支，不代表 GitHub 上一定已经有它。看有没有远程分支，要查 `git branch -a` 或 GitHub 页面。

git status
git add docs/example.md
git commit -m "docs: explain git basics"
git push -u origin feature/git-guide

Git worktree：并行桌面

          
worktree 不是 GitHub 分支。它是本地文件夹，但它可以 checkout 到不同分支。

        

        

          

            
普通切分支

            
一个项目文件夹里切来切去。切到 `main`，这个文件夹显示 main 的文件；切到 `phase4`，同一个文件夹显示 phase4 的文件。

            

              

              
git switch main
git switch feature/phase4-email-autosync

            

          

          

            
常见 worktree

            
多个文件夹并行存在。phase3 和 phase4 可以同时打开，各自有自己的未提交改动，互相不挤占桌面。

            

              

              
git worktree list

/project-main    [main]
/project-phase3  [feature/phase3]
/project-phase4  [feature/phase4]

            

          

        

        

          你的当前总控项目实况：当前分支是 master，没有配置 origin 远程地址，当前只看到一个 worktree。所以这份页面讲的是通用模型，不会假设这个项目已经连上 GitHub。
        

      

    


    

      

        

          
stash：临时抽屉

          
stash 保存的是还没 commit 的改动。它不是分支，也不是 GitHub 备份。

        

        

          

            
什么时候用

            
你写到一半，还不想 commit，但需要切分支、pull、或者让工作区先变干净。

            
这时用 stash 把未完成改动塞进临时抽屉。之后可以拿回来。

            

              

              
git stash push -m "WIP: 正在写 Git 教程"
git stash list
git stash apply
git stash pop

            

          

          

            
apply 和 pop

            
apply 是拿出来，但抽屉里还留一份。适合你怕拿出来失败，想保留备份。

            
pop 是拿出来，并删除抽屉里的那份。适合你很确定这次恢复成功。

            

              

                未提交修改
                已暂存修改
              

              

              

                stash@{0}
                stash@{1}
              

            

          

        

        

          默认 stash 通常不保存新建但还没有被 Git 跟踪的文件。要一起保存新文件，用 git stash push -u -m "message"。
        

      

    


    

      

        

          
merge：把别的版本线合进当前版本线

          
merge 最容易忘的点是“当前分支”。你站在哪条分支上，merge 就会改哪条分支。

        

        

          

            
安全理解

            
如果你想把 feature/git-guide 合进 master，你要先站到 master 上。

            

              

              
git switch master
git merge feature/git-guide

            

            
这句话的意思是：把 feature/git-guide 的改动拿过来，合进我当前所在的 master。

          

          

            
冲突是什么

            
两条分支都改了同一个地方，Git 不知道该保留哪边，就会停下来让你决定。

            
这不是错误，而是 Git 在请你做选择。

          

        


        

          

            
冲突中的文件

            

              
<<<<<<< HEAD

              
教程是本地学习页面。

              
=======

              
教程是交互式学习页面。

              
>>>>>>> feature/git-guide

            

          

          

            
手工解决后

            

              
教程是交互式本地学习页面。

              

              
删除所有冲突标记。

              
保存文件。

              
然后运行 git add 和 git commit。

            

          

        


        

          

          
git status
# 打开冲突文件，手工解决
git add 冲突文件
git commit

# 如果发现合错了，而且还没完成 merge
git merge --abort

        

      

    


    

      

        

          
优秀使用者的 Git 习惯

          
高手不是背了更多命令，而是每次操作前都知道：我站在哪、我要改哪一层状态、如果错了怎么退回来。

        


        

          

            

              
              
              
              
              
              
              
              
            


            

              
Habit 01

              
先看状态，再下命令

              
在敲会改变状态的命令前，先确认当前分支、工作区、暂存区和远端关系。优秀使用者很少靠猜。


              

                

                  他们会问
                  
我现在站在哪条分支？工作区有没有没保存进 commit 的东西？

                

                

                  常见失误
                  
直接 merge、pull、switch，结果才发现自己站错分支或带着脏工作区。

                

                

                  好习惯
                  
先跑只读命令，再决定下一步。看清楚再动手，Git 就不吓人。

                

              


              

                

                
git status
git branch --show-current
git log --oneline --decorate -5

              

            

          

        


        

          

            

              
按 Enter 前的 6 秒自检

              
把这几项养成肌肉记忆，比背十条命令更有用。

            

            

              已检查 0 / 6
              

            

          

          

            
            
            
            
            
            
          

        

      

    


    

      

        

          
你以后先看这张表

          
忘了没关系。先判断你要操作的是工作区、本地历史、worktree，还是 GitHub 远端。

        

        

          

            
              

                你想知道什么
                命令
                它会不会改文件
                怎么理解
              

            
            
              

                现在改了什么
                git status
                不会
                安全检查第一步。
              

              

                当前在哪个分支
                git branch --show-current
                不会
                先确认你站在哪条版本线。
              

              

                有没有 GitHub 远端
                git remote -v
                不会
                没有 origin，就还没连上 GitHub 仓库。
              

              

                Git 开了哪些 worktree
                git worktree list
                不会
                看本地有哪些并行文件夹。
              

              

                保存本地版本
                git add + git commit
                会写入本地历史
                还没有上传 GitHub。
              

              

                上传到 GitHub
                git push
                不会改本地文件
                把本地 commit 发到远程分支。
              

              

                临时收起改动
                git stash push -m "message"
                会让工作区变干净
                把未提交改动放进临时抽屉。
              

              

                合并分支
                git merge 分支名
                会改当前分支
                把对方分支合进你当前站着的分支。
              

            
          

        


        

          

            
先不要随便用

            
git reset --hard、git clean -fd、git push --force 都有丢数据风险。不确定时先让工具或同伴检查。

          

          

            
记忆口诀

            
commit 本地存档，push 上传远端；stash 临时抽屉，merge 合进当前；worktree 是多个本地桌面，PR 是 GitHub 的合并请求。
