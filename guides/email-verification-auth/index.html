Auth Scale / Email Code

邮箱验证码登录：把“能注册”升级成“可验证的账号体系”

这是一份脱敏版接入教程：当一个项目已经有登录注册，但没有邮箱验证码时，按这套最小模块补上发信域名、验证码存储、前端按钮、卡片邮件和部署验证。

它为什么能长成卡片？

验证码邮件不是只能发纯数字。邮件可以是 MIME 消息，里面同时放纯文本和 HTML。邮箱客户端会渲染 HTML，所以你看到的是带颜色、圆角和排版的卡片。

验证码逻辑

服务端生成一次性 6 位数字，保存到数据库或缓存，设置过期时间、已使用状态和发送频率限制。

发信通道

应用通过 Resend API 或 SMTP 登录发信服务。发信服务负责把邮件投递到用户邮箱。

卡片样式

邮件正文使用 HTML 和尽量内联的 CSS。视觉可以像卡片，但安全判断仍只认服务端保存的验证码。

注册登录的最小闭环

以后拿到任何带账号体系的项目，都先把它拆成这 6 步。已有账号密码登录不删，补上邮箱验证这条安全链。

  1. 1
    用户输入邮箱注册页、绑定邮箱页、忘记密码页都可以复用同一个“发送验证码”按钮。
  2. 2
    后端校验邮箱和频率检查格式、同邮箱冷却、同 IP 频率、全局小时上限，避免被刷。
  3. 3
    生成验证码并保存保存邮箱、验证码哈希、用途、过期时间、使用状态和请求上下文。
  4. 4
    通过 Resend 或 SMTP 发出卡片邮件主题写清产品名，正文给出验证码、有效期和安全提醒。
  5. 5
    用户提交验证码注册、重置密码或绑定邮箱时，后端验证用途、邮箱、过期时间和一次性状态。
  6. 6
    消费验证码验证成功后立刻标记已使用，再创建账号、修改密码或绑定邮箱。

手动配置清单

这里用 example.com 作占位符。真实部署时换成自己的域名、邮箱和密钥，不要把密钥写进 Git。

Resend + 域名发信

  • 在 Resend 创建账号，添加发信域名，例如 mail.example.com。
  • 按 Resend 给出的 DNS 记录到 Cloudflare 添加 SPF、DKIM、DMARC 或验证记录。
  • 等待 Resend 显示域名 verified。
  • 创建 API Key，只给当前项目使用。
  • 发件人使用 noreply@mail.example.com，展示名使用产品名。
Vercel 环境变量 .env.example 
SMTP_HOST=smtp.resend.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=resend
SMTP_PASS=<RESEND_API_KEY>
EMAIL_FROM=noreply@mail.example.com
EMAIL_FROM_NAME=Your App
NEXT_PUBLIC_APP_URL=https://app.example.com
ALLOWED_ORIGINS=https://app.example.com

代码应该放在哪几层？

不要把验证码散落在页面组件里。账号体系以后会扩展到注册、登录、重置密码、绑定邮箱，所以要按模块边界放。

层 职责 常见文件名
邮件服务 创建 transporter 或 Resend client，渲染 HTML 和纯文本正文，发送验证码邮件。 lib/email.ts, services/mail.ts
验证码存储 保存哈希、用途、过期时间、使用状态、发送日志和限流计数。 verification_codes, verification_logs
API 路由 提供 send-code、register、reset-password 等接口。 /api/auth/send-code, /api/auth/register
前端表单 在邮箱输入框旁放发送按钮，显示倒计时、错误、成功提示和验证码输入框。 RegisterForm, ForgotPasswordForm
部署配置 配置发信密钥、发件域名、站点 URL、允许来源和生产环境验证命令。 .env.example, Vercel Env, Cloudflare DNS

验证命令

上线后不要只看构建成功。要从正式域名确认页面、预检、业务校验和真实邮件都能跑通。

production-check.sh 替换为自己的域名 
curl -I https://app.example.com/

curl -X OPTIONS -I \
  -A 'Mozilla/5.0 Verification Check' \
  -H 'Origin: https://app.example.com' \
  -H 'Access-Control-Request-Method: POST' \
  https://app.example.com/api/auth/send-code

curl -X POST \
  -A 'Mozilla/5.0 Verification Check' \
  -H 'Content-Type: application/json' \
  -H 'Origin: https://app.example.com' \
  --data '{"email":"bad-email"}' \
  https://app.example.com/api/auth/send-code

不要省掉的安全边界

验证码要有用途、过期时间、一次性消费、频率限制和日志。注册成功后不要继续允许同一个验证码重放；重置密码验证码也不要和注册验证码混用。

公开教程使用脱敏占位符。真实项目应把密钥放进部署平台环境变量，把域名验证记录放进自己的 DNS 服务，并在正式域名上完成端到端验证。
