课程首页 06. 文件生成必须变成真实附件

File Artifacts

生成了，不等于能下载

这一课对应文件产物提交：把 Pyodide 文件、后端工具输出和前端附件卡统一成真实可下载对象。

核心判断

文件生成的完成标准是“OpenWebUI 有文件记录、消息关联了附件、下载接口能返回真实文件”。模型在正文里说完成，只能算过程描述，不能算结果。

四个落点

执行层

Pyodide 扫描 `/mnt/uploads` 的新增文件，连同 MIME 和大小传回主线程。

上传层

主线程把生成物转成浏览器 `File` 并调用 `uploadFile()`。

渲染层

消息组件合并附件来源，并把过程折叠到状态卡。

下载层

文件卡片点击时请求 `?attachment=true`，用响应头文件名触发下载。

验证方式

python -m py_compile backend/open_webui/utils/pdf_generator.py backend/open_webui/tools/builtin.py backend/open_webui/utils/tools.py
node --check static/__fay/openwebui-filegen-status.js

最终验收要看真实附件、真实下载和打开后的文件内容，不只看聊天正文。
