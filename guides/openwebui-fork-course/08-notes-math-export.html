课程首页 08. 笔记捕获、数学渲染和导出

Notes

别让笔记功能漂在 DOM 上

这一课对应笔记提交：把添加到笔记、数学公式和 PDF 导出放回源码，让它们随组件生命周期稳定工作。

核心判断

只要一个功能要长期存在，就应该回到组件和数据模型里。运行时脚本适合临时验证，不适合承载按钮、保存逻辑和可编辑公式。

四个落点

原生入口

聊天消息工具栏直接调用 Notes API，不再扫描 DOM 插按钮。

每日笔记

按日期聚合回答，用 marker 防止重复追加同一条消息。

公式源码

KaTeX HTML 上保存原始 TeX，编辑和导出时都能恢复。

导出排版

打印 CSS 处理分页、表格、代码块和长公式。

验证方式

node --check static/__fay/openwebui-note-export.js
node -e "/* Svelte parse ResponseMessage/RichTextInput/NoteEditor */"

最终要用真实聊天回答测：添加、去重、打开笔记、编辑公式和导出 PDF 都算同一条链路。
