课程首页 07. 图片生成和图片编辑要走意图路由

Image Routing

先判断该不该画，再把图画出来

这一课对应图片路由提交：把图片生成、图片编辑、附件恢复和状态提示连成一个可控闭环。

核心判断

图片功能的第一原则是门控：只有当前轮明确要求生成或编辑图片时才打开图片工具。上下文里有图片，只能说明“可能需要看图”，不能说明“应该画图”。

四个落点

意图路由

把生成、编辑、解释、引用图片拆开判断，避免分析图片时误触发生成。

当前轮输入

编辑图片时只读取当前用户消息里的上传图，不复用历史附件。

附件归一

把 URL、文件 ID、content type 统一成消息可渲染的图片附件。

状态可见

图片生成时显示过程卡，失败时给出清楚的可恢复错误。

验证方式

python fay-custom/scripts/check-image-attachment-normalization.py
python fay-custom/scripts/check-image-context-isolation.py

真正的验收不是“出了一张图”，而是看图、改图、刷新恢复和非图片问题都走到了正确分支。
