课程首页 03. GPT-5.5 模型分级与 Arena 可见性

Model Menu

模型菜单不是前端文案

这一课解释为什么 GPT-5.5 五档模型和 Arena 可见性必须同时处理数据库 preset、权限、全局配置和用户设置。

核心判断

模型下拉要稳定，必须同时改四类数据：模型 preset、普通用户 read 权限、全局 UI 默认值、用户个人 settings。否则当前账号看着正常，换一个普通账号或重启容器就可能丢。

Arena 为什么单独处理

`Arena Model` 是评测配置模型，不完全等同于普通 `model` 表记录。普通用户能否看到它，关键在 `config.evaluation.arena.models[*].meta.access_grants`。默认没有公开读取权限时，管理员能看到，普通用户可能看不到。

四个落点

Preset

五档模型写入 `model` 表，非基座档位指向同一个 `base_model_id`。

权限

`access_grant` 给 `user:*:read`，普通用户才看得到。

默认值

`default_models` 和 pinned/order 列表统一写成五档加 Arena。

用户设置

已有用户 settings 和新用户默认触发器都要更新。

验证方式

python fay-custom/scripts/apply-local-model-tiers.py --data-dir .local-data/open-webui
curl -fsS http://127.0.0.1:8080/api/models -H "Authorization: Bearer $TOKEN"

普通用户也应该看到五档模型和 `arena-model`，而不是只有管理员看到。
