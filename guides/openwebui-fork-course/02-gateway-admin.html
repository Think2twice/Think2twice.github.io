课程首页 02. 网关控制页与上游路由

Gateway Policy

把接口切换做成可控的管理员能力

这一课对应 gateway 管理提交：控制页不直接暴露密钥，刷新只读状态，只有手点测试才触发上游请求。

核心原则

接口切换是管理能力，不是聊天能力。页面刷新必须只是读取状态，不能偷偷测试接口一或接口二。测试上游属于会消耗额度、可能污染上游状态的动作，必须由用户明确点击。

为什么要放进 OpenWebUI 后台

独立 gateway 页面虽然能用，但它绕开了 OpenWebUI 的管理员体系。迁入 `/admin/gateway` 后，浏览器只和 OpenWebUI 后端说话，后端再把管理员 cookie 或 token 转发给 gateway 校验；真实 admin key 可以只留在服务器环境变量或本机私有配置里。

三类请求不要混

普通聊天

代表用户日常对话，可使用 active 上游和自动失败切换。

图片请求

默认优先 `image_preferred`，并且不应改变普通聊天 active。

Embedding

服务 RAG 检索，成功失败也不应该改变普通聊天 active。

管理测试

只有点击“测试”按钮才触发，用于低频排查。

文件对应关系

backend/open_webui/routers/fay_gateway.py
src/lib/apis/fayGateway.ts
src/routes/(app)/admin/gateway/+page.svelte

读代码时按这个顺序看：前端页面发请求，前端 API client 包装，OpenWebUI 后端代理，gateway admin API 执行策略。
