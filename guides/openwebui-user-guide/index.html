ChatGPT 小站使用指南

这份教程不是单纯介绍“怎么聊天”，而是把聊天、笔记、追问、导出和图片生成串成一个完整工作流：先获得答案，再筛选、编辑、补洞，最后沉淀成自己的知识。

先看使用路线 从笔记功能开始

先用这条路线

如果你第一次使用，只要记住这五步：问、收、改、追、导。它的重点不是让聊天变复杂，而是让每次好回答都能留下来。

01

先在 Chat 里提问

选择合适模型，直接提问、上传图片或文件，让 AI 先给出一版答案。

02

把好回答加到笔记

每条 AI 回答下方都有添加到笔记按钮，看到真正有价值的内容就点一下。

03

在笔记里删减改写

笔记是可编辑 Markdown 文本，不是只读收藏夹。删掉废话，保留高密度内容。

04

用 SideChat 继续追问

围绕整篇笔记或选中的一段继续问，回答满意后直接插入正文。

05

导出、分享或回看

笔记和 Chat 都可以复制、下载或导出，适合放进 Notion、微信文件传输助手或本地资料库。

附加能力

图片生成与改图

可以纯文本生成图片，也可以上传图片后要求重新构图、调光影、修风格。

它的官方能力底座

ChatGPT 小站建立在可自托管 AI 平台能力之上，功能覆盖模型接入、RAG、Markdown/LaTeX、图片生成、工具和多端响应式界面。这里的定制体验是在这个底座上继续打磨使用闭环。

模型接入

支持多种模型来源

官方支持 Ollama 与 OpenAI-compatible API，因此可以按实际配置接入不同模型、不同供应商或本地模型。

资料输入

RAG 与文件理解

可以把文档放进聊天或知识库，让 AI 基于文件、网页或检索内容回答。PDF 等文件适合做资料阅读。

表达能力

Markdown 与公式

聊天与笔记都适合承载代码、列表、标题和数学公式，所以学习类、考试类内容不容易散掉。

工具

代码与扩展能力

底层平台支持工具、Python function calling 和插件扩展，适合把模型能力接到实际任务里。

多端

电脑和手机都能用

响应式界面和 PWA 能力让它不局限于桌面端，移动端也可以查看对话、笔记和生成结果。

图片

生成与编辑图片

官方功能本身支持多种图片生成/编辑后端；本实例重点把生成过程和结果展示做得更接近原生体验。

笔记：把散落的好回答收起来

笔记是这套使用方法的核心。它解决的不是“收藏一下”，而是把多个 Chat 里真正有用的回答整理到同一处，慢慢变成一个主题笔记。

添加到笔记像一个每日收集箱

当你第一次点击 AI 回答下方的“添加到笔记”，系统会自动创建一个以当天日期命名的笔记，例如 2026-06-26。之后无论你在哪个 Chat 里继续点击，内容都会追加到这篇日期笔记后面。

当你把这篇日期笔记改名成一个主题，例如“极限与 PPT 总结”或“图片提示词整理”，它就相当于归档成了一个正式笔记。之后再点击“添加到笔记”，系统会重新创建当天日期笔记，开始下一轮收集。

编辑与导出：把回答变成高密度资料

AI 的原始回答经常有冗余。笔记让用户可以直接删掉无用部分、重排结构、保留公式和代码，再导出成自己真正会回看的资料。

这部分适合做“第二遍加工”

你可以先从多个 Chat 里收集三十条有价值回答，再在笔记里删掉重复解释、补上自己的判断，把内容压缩成一篇真正属于自己的高密度笔记。

  • 公式和代码不丢：数学公式、代码块、标题层级在编辑和查看时都能保留较好的格式。
  • 导出方式多：PDF 适合阅读和分享，Markdown 适合导入 Notion 或继续二次编辑，txt 适合纯文本存档。
  • 分页更规整：导出 PDF 时，页面之间会尽量保持清楚分割，便于发到微信文件传输助手或其他设备查看。

SideChat：围绕笔记继续对话

SideChat 是专门服务笔记的侧边对话。它不是另开一个普通 Chat，而是在当前笔记上下文里继续追问、解释、总结，并把有用回答直接插回笔记。

一边看笔记，一边让 AI 补洞

当一篇笔记来自多个 Chat，它很可能还有不连贯、没讲透或你自己还不理解的地方。打开右侧 SideChat，可以直接要求它“总结笔记内容”“解释这一段”“补一个例子”。

打开右侧对话在笔记顶部点击 SideChat 图标，右侧出现对话面板。
围绕整篇笔记提问让 AI 总结、润色、拆结构，适合快速发现笔记里的主线。
把回答插入正文光标放在笔记中某个位置，点击 SideChat 回答旁边的“插入”，回答会进入光标处。
选中文字引用提问选中一段不懂的内容，直接围绕这段问，答案可以插到原文附近。

Chat：完整会话也能管理和回看

如果笔记是精选沉淀，Chat 就是完整过程记录。长对话可以通过导出保存，用右侧大纲快速跳转，也可以选中某段文字后直接引用追问。

长 Chat 用右侧大纲找问题

一个 Chat 里问了很多问题时，手动滚动很容易迷路。右侧大纲会把用户问题整理成列表，平时收起成一个安静的浮条，鼠标移过去后展开。

点击某个问题标题，页面会快速跳转到对应位置。对于学习、排错和长项目讨论来说，这相当于给 Chat 加了一层目录。

选中一段，就围绕这一段继续问

长回答里经常只有某个词、某个选项或某一小段卡住了你。这个时候不需要复制原文重新粘贴，只要选中文字，就可以直接对这段内容提问或要求解释。

这和笔记 SideChat 的引用提问互相呼应：Chat 里适合立刻追问当前回答，笔记里适合围绕沉淀后的资料继续补充。

图片生成：直接生成，也可以按图改图

当前实例接入了高质量图片生成能力。你可以直接用文字生成图片，也可以上传图片，让 AI 重新构图、调整光影、改变风格或做局部改图。

怎么让它更稳定地产出好图

  • 纯文本生成：直接描述主体、构图、风格、用途和比例，例如“做一张冷调建筑感海报，适合小红书封面”。
  • 上传参考图改图：先放图片，再说清楚要改什么，例如“重新构图，让画面更平、光影更好看、保留主体”。
  • 分步调试：第一轮先要方向，第二轮再改光影、裁切、细节和文字，成功率通常比一次说完更高。
  • 注意边界：具体结果取决于当前模型、额度和图片后端配置；复杂局部编辑可以多轮微调。

推荐使用场景

这套功能真正有价值的地方，是减少“复制、粘贴、切应用、回头找”的摩擦，让你能持续整理自己的认知。

学习备考

把多个问题整理成一篇主题笔记

比如围绕极限、网络、算法或英语学习，在不同 Chat 里问到的好解释都追加进同一篇笔记，最后用 AI 命名和 SideChat 补洞。

项目研究

从长对话沉淀决策记录

长 Chat 用大纲定位，关键结论添加到笔记；最后导出 Markdown 或 PDF，形成项目过程资料。

创作与图片

边对话边做视觉方案

可以上传图片让它提取风格，再进一步生成或改图；有用的提示词和结果说明也能加到笔记。

最重要的用法不是“把 AI 当答案机器”，而是把它当成一个持续整理信息、追问疑点、压缩知识密度的工作台。
