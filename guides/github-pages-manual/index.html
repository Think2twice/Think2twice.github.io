教程 归档 GitHub Pages 关于

GitHub Pages 是怎么把文件变成网站的。

这一页从零讲清楚：仓库、分支、根路径、项目路径、CNAME、自定义域名和手动更新流程。你不需要服务器，也不需要自己配证书。

两个最容易混的地址

username.github.io用户主页
username.github.io/repo/项目页面

先分清三种东西

GitHub Pages 不是一台你登录进去的服务器。它更像一个静态文件发布器：你把文件提交到仓库，它把这些文件公开给浏览器访问。

01 Repository

仓库存源码

仓库里放 HTML、CSS、JS、图片和文章。你每次 commit + push，GitHub 都能看到新文件。

02 Pages Source

发布源决定网站内容

Pages 会从某个分支和目录发布，例如 main / root 或 docs/。

03 URL

URL 由仓库类型决定

username.github.io 是根站。普通仓库默认会发布到 /repo-name/ 子路径。

真正手动做一次

        
这是没有任何自动化工具时的最小流程。你只需要 Git、浏览器和 GitHub 账号。

像 Finder 一样看这个博客

        
这个站点没有数据库，也没有后台登录。它就是一组文件夹和 HTML 文件。GitHub Pages 负责把这些文件夹公开成网址。

      

      

        

          

            
            
            
          

          

            
            
            
            
            
            
            
            
            
          

        

        

          Repository
          
整个博客就是一个文件夹

          
GitHub Pages 会把这个仓库里的静态文件发布到 https://think2twice.github.io/。浏览器访问目录时，会默认找这个目录里的 index.html。

          

            
Think2twice.github.io/
├── index.html
├── assets/posts.js
├── guides/
├── archive/
└── about/

          

        

      

    


    

      

        
以后手动新增一篇 HTML 教程

        
最小动作只有两个：把 HTML 放到一个新目录里，再把这篇文章登记到 assets/posts.js。

      

      

        

          

            
在本地准备新教程文件夹

            
每篇教程一个目录，入口文件固定叫 index.html。这样网址会很干净。

            
guides/my-new-guide/
└── index.html

          

        

        

          

            
把你的 HTML 放进去

            
如果你已经有一个完整 HTML，就把它改名或复制成这个目录里的 index.html。

            
cp ~/Desktop/my-guide.html guides/my-new-guide/index.html

          

        

        

          

            
登记到文章列表

            
打开 assets/posts.js，在数组里加一条记录。首页、Recent Posts、分类和归档都会读这里。

            
{
  title: "我的新教程",
  url: "./guides/my-new-guide/",
  archiveUrl: "../guides/my-new-guide/",
  date: "2026-06-20",
  minutes: 8,
  category: "Git",
  tags: ["Git", "教程"],
  featured: false,
  art: "browser",
  excerpt: "一句话说明这篇教程解决什么问题。"
}

          

        

        

          

            
本地预览

            
在仓库根目录起一个静态服务器，打开浏览器检查首页和新教程。

            
python3 -m http.server 8765
# 浏览器打开 http://127.0.0.1:8765/

          

        

        

          

            
提交并发布

            
GitHub Pages 会监听仓库更新。push 之后等一两分钟，线上站点就会更新。

            
git status
git add guides/my-new-guide/index.html assets/posts.js
git commit -m "add my new guide"
git push

          

        

      

    


    

      

        
以后想改模板怎么办

        
这不是 WordPress 那种登录后台点按钮换主题。它是静态站，模板就是 HTML/CSS/JS 文件本身。

      

      

        

          首页模板
          
index.html

          
改首页布局、Recent Posts、分类按钮、侧栏样式，就改根目录的 index.html。

        

        

          文章数据
          
assets/posts.js

          
新增、删除、改标题、分类、标签、摘要，都优先改这里。它相当于一个轻量文章数据库。

        

        

          单篇教程
          
guides/.../index.html

          
每篇教程可以有自己的完整 HTML、CSS 和交互。它们不一定要长得一模一样。

        

      

      

        如果你想要 WordPress 那种“登录后台、新建文章、在线编辑 Markdown”的体验，可以换成更完整的静态站方案，比如 Astro / VitePress / Docusaurus + GitHub Pages，或者接一个 CMS。但当前这个纯 HTML 方案优点是最简单、免费、快、不需要服务器。
      

    


    

      

        
根站和项目站怎么选

        
你现在看到的旧博客在根路径，是用户主页仓库。新的教程也可以直接放根站里，做成一个统一的学习博客。

      

      

        

          用户主页
          
username.github.io

          
仓库名必须是 username.github.io。它发布到根路径，最适合做个人主页、博客入口、教程库首页。

        

        

          项目页面
          
username.github.io/repo/

          
普通仓库会发布到仓库名子路径。适合单个项目文档，但教程多了以后入口会变分散。

        

        

          自定义域名
          
CNAME

          
CNAME 会让 GitHub Pages 改用你的域名。旧域名失效时，要删掉 CNAME 或在设置里清空域名。

        

      

    


    

      

        最省心的长期方案：
        把根站做成教程博客入口。以后每篇交互 HTML 放到 guides/教程名/index.html，再在首页文章元数据里加一条记录，Recent Posts、分类和归档就会自动显示。

    let current = 0;

    function renderTokens(target, items, color) {
      target.innerHTML = "";
      if (!items.length) {
        const empty = document.createElement("span");
        empty.className = "empty";
        empty.textContent = "空";
        target.appendChild(empty);
        return;
      }
      items.forEach((item) => {
        const token = document.createElement("span");
        token.className = "token";
        token.style.setProperty("--color", color);
        token.textContent = item;
        target.appendChild(token);
      });
    }

    function render() {
      const step = steps[current];
      els.command.textContent = step.command;
      els.output.textContent = step.output;
      els.label.textContent = step.label;
      els.lesson.textContent = step.lesson;
      els.back.disabled = current === 0;
      els.next.disabled = current === steps.length - 1;
      els.next.textContent = current === steps.length - 1 ? "已完成" : "下一步";
      renderTokens(els.local, step.local, "var(--coral)");
      renderTokens(els.repo, step.repo, "var(--teal)");
      renderTokens(els.pages, step.pages, "var(--blue)");
      renderTokens(els.url, step.url, "var(--green)");
    }

    els.next.addEventListener("click", () => {
      current = Math.min(steps.length - 1, current + 1);
      render();
    });
    els.back.addEventListener("click", () => {
      current = Math.max(0, current - 1);
      render();
    });
